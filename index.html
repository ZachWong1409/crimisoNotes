Apriori — Association Rules

unsupervised · co-occurrence
Formula recall
minsup count = ⌈minsup% × total transactions⌉
support(X) = count(X) / total transactions
confidence(X→Y) = count(X∪Y) / count(X)
  1. Count 1-itemsets → prune → L1
  2. Pair → count 2-itemsets → prune → L2
  3. Join L2 (shared items + frequent subsets only) → L3
  4. Generate rules, keep confidence ≥ threshold
Extra for Q5
Lift = Confidence(X→Y) / Support(Y)
Leverage = Support(X∪Y) − [Support(X)×Support(Y)]

Apriori principle: if an itemset is frequent, all its subsets must be frequent too (anti-monotone support) — this is why pruning works.

Lift>1 = real association · Lift=1 = coincidence · Lift<1 = negative association

Application: Market basket analysis (which products sell together), recommendation engines ("customers who bought X also bought Y"), fraud/ping-pong detection in healthcare, web click-pattern analysis.

Decision Tree — ID3

supervised · classification
Formula recall
H = −Σ p(class)·log₂p(class)
H(attribute) = Σ (branch size/total) × H(branch)
Gain = H(parent) − H(attribute)
  1. Base entropy of full dataset
  2. Gain for every attribute
  3. Split on highest Gain
  4. Recurse inside each branch until pure
Extra for Q5

Algorithms: ID3 (Info Gain) · C4.5 (Gain Ratio, fixes ID3 bias) · CART (Gini Index, does regression too)

Continuous target? Use a Regression Tree — same splits, but leaf = mean of target value, not majority vote.

Overfitting fix: Pruning — pre-prune (stop early) or post-prune (trim after full growth).

Application: Customer churn prediction, credit risk approval, medical diagnosis decision support, loan default classification.

Naive Bayes

supervised · probabilistic · generative
Formula recall
Pick class maximizing:
P(class) × ∏ P(value|class)
P(value|class) = count(value∧class) / class total
Laplace (only if count=0):
(count+1) / (class total + distinct values of that attribute)

⚠ Denominator = class size, never attribute total. Smooth only the zero value, not the whole attribute.

Extra for Q5

3 types: Gaussian (continuous), Multinomial (discrete counts — text/word freq), Bernoulli (binary present/absent)

"Naive" = assumes all features are conditionally independent given the class (rarely true, works anyway)

Generative model — learns P(features|class) then flips via Bayes' theorem, unlike discriminative models (Decision Tree, Logistic Regression)

Application: Spam email filtering (word frequency → spam/not spam), sentiment analysis, medical diagnosis from symptoms, document/topic categorization.

K-Means Clustering

unsupervised · centroid-based
Formula recall
d = √[(x₁−x₂)² + (y₁−y₂)²]
  1. Start with k initial centroids
  2. Assign each point to nearest centroid
  3. Recompute centroid = mean(x), mean(y) of members
  4. Repeat 2–3 until assignments stop changing
Extra for Q5

Advantages: simple, fast, scales well

Disadvantages: must pick k upfront, sensitive to initial centroids, sensitive to outliers, assumes spherical clusters

Elbow method: plot WCSS vs k, pick the k where the curve bends ("elbow") — diminishing returns past that point

Application: Customer segmentation for marketing, image compression (group similar pixel colors), document/topic clustering, anomaly detection (points far from any centroid).

Don't trip on these

Apriori: round support count up, never down
Naive Bayes: divide by class total, not attribute total
Decision Tree: a 2-2 split has entropy 1, not 0
K-Means: stop on stable assignment, not fixed rounds
Apriori: never join itemsets with an infrequent subset
Naive Bayes: smooth only the zero-count value, nothing else
